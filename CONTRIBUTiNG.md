# Comment contribuer à Skynote

Merci de votre intérêt pour contribuer à Skynote ! 🎉

## 🐛 Reporter un bug

1. Vérifier qu'il n'existe pas déjà dans les issues
2. Créer une nouvelle issue avec :
   - Description claire du problème
   - Étapes pour reproduire
   - Comportement attendu
   - Screenshots si applicable

## 💡 Proposer une fonctionnalité

1. Ouvrir une discussion dans les issues
2. Décrire clairement la fonctionnalité
3. Expliquer pourquoi elle serait utile

## 🔧 Soumettre un Pull Request

1. Fork le projet
2. Créer une branche (`git checkout -b feature/AmazingFeature`)
3. Commit les changements (`git commit -m 'Add AmazingFeature'`)
4. Push la branche (`git push origin feature/AmazingFeature`)
5. Ouvrir un Pull Request

## 📝 Standards de code

- Utiliser des noms de variables clairs
- Commenter le code complexe
- Suivre le style existant
- Tester sur différents navigateurs

## 📋 Code de conduite

- Être respectueux et inclusif
- Accepter les critiques constructives
- Se concentrer sur ce qui est le mieux pour la communauté
